* Introducción
Este documento detalla la configuración técnica y el flujo de trabajo del proyecto *Quizs-Evo*, una plataforma web para la gestión de alumnos y cuestionarios basada en tecnologías *serverless* (Firebase).

* 1. Infraestructura en la Nube (Firebase)
** 1.1. Creación del Proyecto
1. Acceder a [[https://console.firebase.google.com/][Firebase Console]].
2. Crear un nuevo proyecto: =quizs-evo=.
3. Registrar una *Aplicación Web* (icono </>) para obtener las credenciales del SDK.

** 1.2. Activación de Servicios
*** Firebase Authentication
- **Función:** Gestiona el acceso (login/password).
- **Configuración:** Habilitar el proveedor "Correo electrónico/contraseña".

*** Cloud Firestore (Base de Datos)
- **Función:** Almacena la lista de usuarios para la interfaz del Admin.
- **Configuración:** Crear en "Modo de prueba".
- **Reglas de Seguridad:**
#+begin_src javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
#+end_src

* 2. Código Fuente (index.html)
El sistema utiliza módulos de Firebase (v10.7.1) cargados vía CDN.

** 2.1. Configuración del SDK y Base de Datos
#+begin_src javascript
import { initializeApp } from "https://www.gstatic.com/firebasejs/10.7.1/firebase-app.js";
import { getAuth } from "https://www.gstatic.com/firebasejs/10.7.1/firebase-auth.js";

    apiKey: "AIzaSy...",

** 2.2. Importación Masiva desde CSV
Se utiliza la librería =PapaParse= para procesar archivos locales y subirlos a la nube.
#+begin_src javascript
// Bucle de procesamiento de filas
for (let usuario of results.data) {
    if(usuario.email && usuario.password) {
        // Registro en Authentication (Acceso)
        await createUserWithEmailAndPassword(auth, usuario.email, usuario.password);
        
        // Registro en Firestore (Tabla visual)
        await setDoc(doc(db, "usuarios", usuario.email), {
            email: usuario.email,
            rol: "alumno"
        });
    }
}
