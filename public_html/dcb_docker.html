1. Info

Docker-Controller-Bot es un bot de Telegram que permite controlar tus contenedores Docker remotamente desde un único lugar. Proporciona comandos para listar, arrancar, detener y eliminar contenedores, ver logs, extraer el docker-compose de contenedores y recibir notificaciones de estado o actualizaciones pendientes. También puede actualizar contenedores, cambiar la etiqueta de las imágenes y limpiar recursos no utilizados. El bot se configura mediante variables de entorno (como TELEGRAMTOKEN y TELEGRAMADMIN) y accede al socket de Docker para gestionar el motor local de contenedores. Es útil para administradores que desean gestionar Docker directamente desde Telegram.

2. Uso

  • Control del homelab desde Telegram
  • Gestión básica sin SSH

3. Enlace

https://github.com/dgongut/docker-controller-bot

4. Variables de entorno

TELEGRAMTOKEN
Token del bot
TELEGRAMADMIN
ID de telegram del administrador
CHECKUPDATEEVERYHOURS
Frecuencia de actualización

5. docker-compose.yml

services:
    docker-controller-bot:
        environment:
            - TELEGRAM_TOKEN=*****************************
            - TELEGRAM_ADMIN=********
            - CONTAINER_NAME=docker-controller-bot
            - TZ=Europe/Madrid
            - CHECK_UPDATE_EVERY_HOURS=8
            - LANGUAGE=ES
        volumes:
            - /var/run/docker.sock:/var/run/docker.sock # NO CAMBIAR
            - /home/sherlockes/dockers/dcb:/app/schedule # CAMBIAR LA PARTE IZQ>
        image: dgongut/docker-controller-bot:latest
        container_name: docker-controller-bot
        restart: always
        network_mode: host
        tty: true

6. Gestión

docker compose up -d
docker compose logs -f docker-controller-bot
